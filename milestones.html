3-Month Adaptive Python & Data Structure Blueprint (Pre-Semester Prep)

Hey team! Welcome to your 12-week study roadmap for ICT Batch 681, designed to be completed before the semester starts. If you struggle to memorize syntax or freeze up on blank screens, this is for you. We use a friendly "Read, Break, and Fix" style so you build real programming logic without stressing your memory storage.

Brain-Friendly Learning Rules (No Memorization Required)

To fit this training into 3 months without burning out your brain networks, follow these three rules strictly:

1. Keep Cheat Sheets Open: Never type code from pure memory. Keep your reference guides open on your screen at all times.
2. Tweak One Single Line: When working with a project snippet, change exactly one variable or value at a time, run it, and look at what changes.
3. Guess the Error: When the code breaks, don't panic. Read the error line carefully like a puzzle hint to see exactly where the computer got confused.

Month 1: Core Syntax & Algorithmic Control Flow (Weeks 1-4)

Get comfortable with core code mechanics, variable boxes, basic types, and logical execution tracks.

Timeline Weekly Template Project Target Objectives (Read, Modify, Write Instructions)
Week 1 Project: Currency Converter 1. Open up a working script template that displays text and numbers on your screen.
2. Tweak the numbers inside the variable slots to see how the math recalculates automatically.
3. Add your own customized text labels inside the print brackets using pair quotes.
4. Break the code on purpose by deleting a variable name to check how Python alerts you.
Week 2 Project: ATM PIN Validator 1. Read through a template script that uses if, elif, and else blocks to choose directions.
2. Edit the target check numbers to force the code down a completely different logic path.
3. Add a fresh decision layer by dropping a new elif block right into the template code.
4. Connect two checking statements together inside an if rule using the 'and' or 'or' keywords.
Week 3 Project: Number Guessing Game 1. Load up a loop template that runs repeatedly until a specific tracking variable hits a boundary.
2. Change the starting variable value to see how it makes the loop run shorter or longer.
3. Drop a break keyword straight inside the running loop to force it to stop instantly.
4. Adjust the loop conditions to a rule that can never be met to observe how an infinite loop acts.
Week 4 Project: Grid Coordinator Map 1. Look over a standard for loop running inside a fixed numeric range statement.
2. Modify the range boundary numbers to change where the loop count starts and stops.
3. Study a nested loop layout that prints a multi-line matrix coordinate grid map.
4. Insert an if rule inside the loop body to filter out and skip printing specific odd numbers.

Month 2: Structural Data Collections & OOP Systems (Weeks 5-8)

Transition from loose variables into organized collection layouts and reusable object-oriented code blocks.

Timeline Weekly Template Project Target Objectives (Read, Modify, Write Instructions)
Week 5 Project: Student Grade Book Ledger 1. Read an existing list data collection pre-loaded with a row of sample student grades.
2. Use append and pop methods to drop fresh scores in or pull old scores out of the list.
3. Edit the index numbers inside the square brackets to grab a custom subset slice of the data list.
4. Try to alter an item value inside an unchangeable tuple to see the data type protection lock activate.
Week 6 Project: Inventory Stock Directory 1. Look at a key-value dictionary block that maps store items directly to their price values.
2. Change a product price value by calling its text key label directly inside square brackets.
3. Drop a brand new item key and price pair entry into the existing dictionary variable.
4. Run a loop template that prints keys and values side-by-side using the items function.
Week 7 Project: Bank Account System 1. Scan through a class blueprint containing property variables and internal function behaviors.
2. Alter the input settings inside the __init__ builder function to change an object's starting parameters.
3. Insert a print message inside an operational method to watch data shifts happen in real time.
4. Spin up a separate, second instance of the class to see how both objects hold isolated details.
Week 8 Project: Retail Checkout Tree 1. Study a sub-class code block that automatically inherits fields from a parent class structure.
2. Edit a method inside the sub-class to change how it overrides the parent's default behavior.
3. Lock down an internal property variable by adding double underscores to make it private.
4. Use a helper getter function to read the hidden attribute safely without triggering a crash.

Month 3: Data Structures & Algorithms (DSA) with Python Course (Weeks 9-12)

Master advanced computer algorithms, evaluate execution speeds using Big-O rules, and learn how pointer structures connect data arrays under the hood.

Timeline Weekly DSA Python Project Target Objectives (Read, Modify, Write Instructions)
Week 9 Project: Array Memory Tracker & Linear Search 1. Open a template script that measures how many operations a Linear Search takes to find a number in a list.
2. Swap the list with a sorted number set, run a Binary Search script, and look at how the number of steps drops dramatically.
3. Print out your tracking index values inside the loop to calculate operational performance efficiency rules manually.
4. Change target search values to items that do not exist to observe how algorithms signal a missing item exception.
Week 10 Project: Advanced Comparison Sorting 1. Read a Merge Sort template that cuts an unsorted list completely in half, sorts the small pieces, and glues them back together.
2. Tweak the numbers inside the list to see how Merge Sort always takes a steady, predictable amount of time to finish.
3. Load a Quick Sort script template that uses a single 'pivot' number to split lower items to the left and higher items to the right.
4. Change the pivot selection from the 'first item' to the 'middle item' to observe how it changes the sorting speed on a messy data set.
Week 11 Project: Binary Heaps & Heap Sort 1. Open a structural template that simulates a Binary Heap layout (organizing values like a tree where parents are always larger than their child nodes).
2. Tweak individual values inside the heap list to watch how a 'heapify' script automatically bubbles numbers up or down to repair the tree order.
3. Run a Heap Sort script template that constantly pulls the largest top number off the heap to build a perfectly sorted list.
4. Contrast the performance of Heap Sort against Merge Sort to verify why both are considered advanced, comparison-based sorting paths.
Week 12 Project: Linked Lists & Processing Pipelines 1. Examine a Python class that builds a custom Node object holding an asset value alongside a next pointer reference.
2. Modify an intermediate pointer memory link to drop a middle node cleanly out of a Singly-Linked List without shifting array items.
3. Load a stack tracking tool template constrained by Last-In-First-Out (LIFO) operational rules.
4. Process data variables using enqueue and dequeue lines inside a First-In-First-Out (FIFO) queue pipeline sequence.
